# Bloomberg

> Bloomberg delivers business and markets news, data, analysis, and video to the world. For developers, Bloomberg provides the Open API (BLPAPI) — a service-oriented, socket-based API used by the Desktop API, Server API (SAPI), B-PIPE, and Bloomberg Platform products — plus Data License content delivery and the EMSX trading API. BLPAPI is not natively HTTP/REST: clients use the SDK (C++, Java, .NET, Python, Node.js) over a TCP session, typically localhost:8194 for the Desktop API.

## APIs

- [Bloomberg Market Data API](https://www.bloomberg.com/professional/support/api-library/): Real-time and historical market data — stocks, indices, commodities, currencies.
- [Bloomberg Server API (SAPI)](https://www.bloomberg.com/professional/products/data/data-connectivity/server-api/): Terminal data for server applications.
- [Bloomberg Data License API](https://www.bloomberg.com/professional/products/data/data-management/data-license/): Reference, pricing, regulatory, ESG, corporate actions, fundamentals, and alternative data (HTTP delivery via api.bloomberg.com, auth required).
- [Bloomberg EMSX API](https://www.bloomberg.com/professional/products/trading/execution-management-system/): Programmatic equities, futures, and options order management ([docs](https://emsx-api-doc.readthedocs.io/)).
- [Bloomberg BLPAPI Core](https://www.bloomberg.com/professional/support/api-library/): The foundational Open API — Request/Response, Subscription, and Publishing paradigms across //blp/refdata, //blp/mktdata, //blp/mktbar, //blp/mktvwap, //blp/apiflds and more.

## Auth

- BLPAPI: session-based — Desktop API inherits the logged-in Bloomberg Professional (Terminal) identity on localhost:8194; SAPI/B-PIPE authorize users/apps via the //blp/apiauth service (AuthorizationRequest, Identity, AuthOptions).
- Web/console + Data License HTTP delivery: OAuth 2.0 / OIDC — discovery published at https://www.bloomberg.com/.well-known/oauth-authorization-server (endpoints on login.bloomberg.com, authorization_code + refresh_token, PKCE S256, scopes: openid, user, entitlements).
